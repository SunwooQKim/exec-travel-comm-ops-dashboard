# Contributing

Thank you for your interest in improving this project.

## Workflow

1. Fork the repository (if you do not have direct write access).
2. Create a branch from the default branch: `git checkout -b feature/your-change`.
3. Make focused commits with clear messages.
4. Open a pull request describing the problem, the approach, and any test or manual verification you performed.

## Standards

- Match existing **TypeScript** and **React** style in the codebase.
- Run `npm run build` before submitting when you touch `packages/core` or `apps/web`.
- Avoid committing secrets; use `.env.example` only for documenting variables.

## Scope

This is primarily a **portfolio / demo** codebase. Large refactors or new features are welcome as proposals in the PR description so reviewers can align on direction.
